# Agent Manifests: встраивать в промпт или держать отдельно?

**Дата:** 2026-03-16
**Контекст:** Архитектурное решение по организации субагентов в проекте

---

## Что такое манифест агента

Манифест — это детальное описание воркфлоу, чеклистов, шаблонов и правил работы агента. Пример: `docs/manifests/agents/devops.md` (400+ строк).

Промпт агента — это файл `.claude/agents/<name>.md` с frontmatter + системные инструкции. Это то, что Claude Code загружает при вызове субагента.

**Вопрос:** куда класть детальный воркфлоу — прямо в промпт или отдельным файлом?

---

## Подход 1: Всё в промпте (inline)

```
.claude/agents/devops.md    ← 500+ строк: frontmatter + инструкции + воркфлоу + чеклисты
```

### Плюсы
- Агент получает 100% контекста сразу при вызове
- Нет риска "забыть прочитать" манифест
- Один файл — проще поддерживать
- Не тратится turn на чтение дополнительных файлов

### Минусы
- Промпт раздувается (devops = 400+ строк воркфлоу)
- Дублирование при обновлении (промпт + документация)
- Не масштабируется: 10 агентов × 400 строк = хаос
- Нет отдельного версионирования
- Сложнее шарить между проектами

---

## Подход 2: Отдельные манифесты (наш текущий подход)

```
.claude/agents/devops.md           ← ~60 строк: frontmatter + краткие инструкции + ссылка на манифест
docs/manifests/agents/devops.md    ← 400+ строк: детальный воркфлоу, чеклисты, шаблоны
docs/manifests/agents/devops.yaml  ← метаданные, зависимости, конфигурация
```

### Плюсы
- Промпт лёгкий и читабельный
- Манифест можно обновлять без правки промпта
- Версионирование (yaml с metadata, version, updated_at)
- Можно шарить манифесты между проектами (X0 Framework → проект)
- Масштабируется (30+ агентов без хаоса)
- Разделение ответственности: "кто я" (промпт) vs "как я работаю" (манифест)

### Минусы
- Агент должен сам прочитать манифест (тратит 1 turn на Read)
- Если агент забудет прочитать — потеряет контекст
- Два-три файла вместо одного
- Зависимость от file read permissions

---

## Подход 3: Гибрид через skills (продвинутый)

Claude Code поддерживает поле `skills` в frontmatter субагента. Skills **предзагружаются** в контекст автоматически — агент получает их содержимое без отдельного Read.

```yaml
---
name: devops
description: DevOps Engineer для деплоя и инфраструктуры
skills:
  - devops-workflow    # контент загрузится автоматически при вызове агента
---
```

Для этого манифест нужно оформить как skill (`.claude/skills/devops-workflow.md`), а не как обычный `.md` в `docs/`.

### Плюсы
- Агент получает контекст автоматически (как inline)
- Манифест отдельным файлом (как подход 2)
- Не тратится turn на Read
- Нет риска "забыть прочитать"

### Минусы
- Нужно дублировать манифест как skill
- Skills имеют свой формат (не yaml metadata)
- Сложнее синхронизировать с X0 Framework

## Сравнительная таблица

| Критерий | Inline | Отдельно | Гибрид (skills) |
|----------|--------|----------|-----------------|
| Контекст сразу | ✅ | ❌ (нужен Read) | ✅ |
| Масштабируемость | ❌ | ✅ | ✅ |
| Версионирование | ❌ | ✅ (yaml) | ❌ |
| Шаринг между проектами | ❌ | ✅ | Частично |
| Простота | ✅ | Средняя | ❌ |
| Кол-во файлов на агента | 1 | 2-3 | 2 |
| Совместимость с X0 | ❌ | ✅ | Частично |

## Наше решение для StatusPulse

**Подход 2 (отдельные манифесты)** с инструкцией в промпте "прочитай манифест перед работой".

Причины:
1. Совместимость с X0 Framework (манифесты копируются as-is)
2. Масштабируемость (планируем добавлять агентов)
3. Чёткое разделение: промпт (кто я + какие tools) vs манифест (как работать)

### Структура файлов

```
.claude/agents/devops.md              ← Промпт + frontmatter (name, description, model, tools)
                                         Содержит: краткие инструкции + "прочитай манифест!"

docs/manifests/agents/devops.md       ← Детальный манифест (воркфлоу, чеклисты, шаблоны)
                                         Источник: X0 Framework

docs/manifests/agents/devops.yaml     ← Метаданные (version, capabilities, dependencies)
                                         Источник: X0 Framework
```

### Как это работает

1. Claude Code вызывает субагента `devops`
2. Агент загружается с промптом из `.claude/agents/devops.md`
3. В промпте написано: "Прочитай `docs/manifests/agents/devops.md` перед работой"
4. Агент читает манифест (1 turn) и получает полный воркфлоу
5. Агент выполняет задачу по манифесту

---

## Возможная эволюция

В будущем можно перейти на **гибрид (skills)**, если:
- Количество агентов вырастет до 5+
- Потеря turn на Read станет критичной
- X0 Framework добавит поддержку skills-формата
